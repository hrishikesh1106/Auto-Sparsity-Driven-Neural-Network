## ‚úāÔłŹ The Core Idea

</div>

> **Traditional pruning:** Train ‚Üí Evaluate ‚Üí Prune ‚Üí Fine-tune *(4 steps, post-hoc)*
>
> **This approach:** Train *(1 step ‚ÄĒ pruning happens automatically, from epoch 1)*

Every single weight in this network has a **learnable gate** ‚ÄĒ a sigmoid scalar that can open or close the connection. An L1 sparsity penalty creates constant pressure to close gates. The classification loss fights back, keeping gates open only when they genuinely matter. The result? The network **learns which connections to kill**, producing a lean, efficient model without any post-training intervention.

## ūüĒ¨ How It Works

</div>

### Part 1 ‚ÄĒ `PrunableLinear`: The Gated Layer

A custom `nn.Module` that replaces every `nn.Linear` in the network. It holds **three learnable parameter tensors**:

| Parameter | Shape | Purpose |
|:---:|:---:|:---|
| `weight` | `(out_features, in_features)` | Standard connection weights |
| `bias` | `(out_features,)` | Standard bias term |
| `gate_scores` | `(out_features, in_features)` | Raw logits for each gate ‚ÄĒ **learned by the optimizer** |

**The forward pass in 3 lines:**
```python
gates          = torch.sigmoid(self.gate_scores)   # ‚Üí values in (0, 1)
pruned_weights = self.weight * gates               # ‚Üí element-wise mask
return F.linear(x, pruned_weights, self.bias)      # ‚Üí standard affine
```

Gradients flow to **both** `weight` and `gate_scores` automatically ‚ÄĒ all ops are native PyTorch primitives, no custom `backward()` required.

<br/>

### Part 2 ‚ÄĒ Sparsity Loss: Why L1?

```
Total Loss = CrossEntropy(logits, labels)  +  őĽ ¬∑ ő£ŠĶĘ sigmoid(gate_scoreŠĶĘ)
```

The penalty term is the **L1 norm of all gate values**. Here's why L1 (not L2) is the right choice:

```
‚ąā(L1) / ‚ąā(gate_score) = sign(gate) = +1   always (gates > 0 after sigmoid)
```

| Regularizer | Gradient near zero | Effect |
|:---:|:---:|:---|
| **L2** | `2g ‚Üí 0` as `g ‚Üí 0` | Shrinks gates but rarely reaches *exactly* zero |
| **L1** | `+1` *always* | Constant pull ‚ÄĒ reliably drives gates to **exactly** zero |

The L1 penalty creates a tug-of-war: the classification loss keeps important gates open; the sparsity loss kills everything else.

<br/>

### Part 3 ‚ÄĒ Training Loop

```python
# Single backward pass updates weights AND gates simultaneously
ce_loss = F.cross_entropy(logits, labels)
sp_loss = sparsity_loss(model)            # L1 on all sigmoid(gate_scores)
loss    = ce_loss + lam * sp_loss
loss.backward()
optimizer.step()
```

After training, any weight whose gate drops below `0.01` is considered **pruned**.
